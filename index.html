Manifiesto

El Fin de los Silos de la Ingeniería

La inteligencia, durante las primeras décadas de la computación, fue un fantasma atrapado en servidores lejanos. Pero la verdadera revolución de la IA no ocurre en la nube; ocurre cuando el algoritmo toca el barro, el metal y el silicio.

Este tratado nace de una premisa fundamental: el software y el hardware han dejado de ser disciplinas separadas para convertirse en un único tejido ciberfísico. Ya no podemos diseñar un material sin pensar en cómo sus datos de fatiga serán leídos por sensores; no podemos programar un microcontrolador sin entender las limitaciones térmicas de los polímeros que lo rodean; y no podemos trazar una norma ISO sin saber que una IA generativa está a punto de redefinir la topología de esa pieza.

"La ingeniería del siglo XXI no dibuja planos: cultiva organismos. No escribe código: entrena instintos. No selecciona materiales: negocia con la termodinámica." — E. E. Bonillo

Capítulo 01 · Parte I

Del Interruptor a la Red Neuronal

Los sensores ya no son meros interruptores: son receptores sensoriales que alimentan redes neuronales con tensores de datos del mundo físico. Este capítulo establece el puente entre la electrónica clásica y el Machine Learning moderno.

1.1 Fundamentos Físicos del Sensor

Un sensor es un transductor: convierte una magnitud física en una señal eléctrica mensurable. La relación entre la magnitud de entrada x y la señal de salida y se describe mediante la función de transferencia:

y(t) = S · x(t) + y₀ + ε(t) Ec. 1.1 — Modelo lineal de un sensor, donde S es la sensibilidad, y₀ el offset y ε el ruido

Donde:

  • S (sensibilidad): pendiente de la curva de calibración [V/Pa, mV/°C, etc.]
  • y₀ (offset): señal de salida cuando x = 0
  • ε(t) (ruido): componente estocástica, típicamente modelada como gaussiana 𝒩(0, σ²)

1.2 El Sensor como Neurona Periférica

En el paradigma de la IA física, un sensor es análogo a una neurona sensorial. Considere un acelerómetro MEMS de 6 ejes (MPU-6050):

  • Frecuencia de muestreo: 100 Hz (teorema de Nyquist-Shannon)
  • 6 valores por muestra: aₓ, aᵧ, a_z, ωₓ, ωᵧ, ω_z
  • Flujo de datos: 600 valores/segundo = potencial de acción
x(t) = [aₓ, aᵧ, a_z, ωₓ, ωᵧ, ω_z]ᵀ ∈ ℝ⁶ Ec. 1.2 — Vector de estado de un IMU de 6 grados de libertad

1.2.1 Extracción de características (Feature Engineering)

Las redes neuronales consumen features, no datos crudos. Para una ventana temporal de N muestras, extraemos:

C++17// Extracción de características estadísticas en C++ moderno
#include <array>
#include <cmath>
#include <numeric>
#include <algorithm>

template<typename T, size_t N>
struct FeatureVector {
    T mean, std_dev, rms, peak, peak_to_peak;
    T skewness, kurtosis, zero_crossings;
    T energy, entropy;

    static FeatureVector compute(const std::array<T, N>& window) {
        FeatureVector fv;

        // Media
        fv.mean = std::accumulate(window.begin(), window.end(), T(0)) / N;

        // Desviación estándar
        T sq_sum = std::accumulate(window.begin(), window.end(), T(0),
            [&](T acc, T v) { return acc + (v - fv.mean) * (v - fv.mean); });
        fv.std_dev = std::sqrt(sq_sum / N);

        // RMS (Root Mean Square)
        T sq_sum_raw = std::accumulate(window.begin(), window.end(), T(0),
            [](T acc, T v) { return acc + v * v; });
        fv.rms = std::sqrt(sq_sum_raw / N);

        // Peak y Peak-to-Peak
        auto [min_it, max_it] = std::minmax_element(window.begin(), window.end());
        fv.peak = *max_it;
        fv.peak_to_peak = *max_it - *min_it;

        // Skewness (asimetría)
        T m3 = std::accumulate(window.begin(), window.end(), T(0),
            [&](T acc, T v) { return acc + std::pow(v - fv.mean, 3); }) / N;
        fv.skewness = m3 / std::pow(fv.std_dev, 3);

        // Kurtosis (aplastamiento)
        T m4 = std::accumulate(window.begin(), window.end(), T(0),
            [&](T acc, T v) { return acc + std::pow(v - fv.mean, 4); }) / N;
        fv.kurtosis = m4 / std::pow(fv.std_dev, 4) - 3.0;

        // Zero crossings
        fv.zero_crossings = 0;
        for (size_t i = 1; i < N; ++i)
            if ((window[i] >= 0) != (window[i-1] >= 0)) fv.zero_crossings++;

        // Energía de la señal
        fv.energy = sq_sum_raw;

        return fv;
    }
};
📐 Fundamento Matemático

Skewness: γ₁ = μ₃ / σ³ mide la asimetría de la distribución.

Kurtosis: κ = μ₄ / σ⁴ − 3 mide el "peso" de las colas. Una kurtosis alta indica presencia de picos (útil para detectar fallos en rodamientos).

1.4 Condicionamiento de Señal

Antes de la digitalización (ADC), la señal analógica requiere acondicionamiento:

V_out = G · (V_sensor − V_ref) + V_offset Ec. 1.3 — Etapa de acondicionamiento con amplificador de instrumentación

Las etapas típicas son:

  1. Amplificación: ganancia G adaptada al rango del ADC (típicamente 3.3 V o 5 V)
  2. Filtrado paso-bajo: eliminación de ruido de alta frecuencia (filtro RC o activo Butterworth)
  3. Offset: desplazamiento del nivel DC para centrar la señal
  4. Protección: diodos de clamp para evitar saturación del ADC

1.4.1 Filtro anti-aliasing (Nyquist-Shannon)

El Teorema de Muestreo de Nyquist-Shannon establece que la frecuencia de muestreo fₛ debe cumplir:

fₛ ≥ 2 · f_max Ec. 1.4 — Criterio de Nyquist para evitar aliasing

En la práctica, se usa un factor de seguridad de 2.5× a 4× y un filtro pasa-bajos de orden n con frecuencia de corte f_c = fₛ / 2.5.

C++17// Filtro digital IIR Butterworth de 2º orden (biquad)
// Coeficientes calculados con scipy.signal.butter(2, fc/fs)
class ButterworthLPF {
    double b0, b1, b2, a1, a2;
    double x1 = 0, x2 = 0, y1 = 0, y2 = 0;

public:
    ButterworthLPF(double fc, double fs) {
        double omega = 2.0 * M_PI * fc / fs;
        double c = 1.0 / std::tan(omega / 2.0);
        double denom = c*c + std::sqrt(2.0)*c + 1.0;
        b0 = 1.0 / denom;
        b1 = 2.0 * b0;
        b2 = b0;
        a1 = 2.0 * (c*c - 1.0) / denom;
        a2 = (c*c - std::sqrt(2.0)*c + 1.0) / denom;
    }

    double filter(double x0) {
        double y0 = b0*x0 + b1*x1 + b2*x2 - a1*y1 - a2*y2;
        x2 = x1; x1 = x0;
        y2 = y1; y1 = y0;
        return y0;
    }
};

2.3 Ley de Amdahl Aplicada al Edge

La Ley de Amdahl establece el speedup máximo al paralelizar una tarea:

S(N) = 1 / ((1 − P) + P/N) Ec. 2.1 — Ley de Amdahl, donde P es la fracción paralelizable y N el número de núcleos

En Edge Computing, la fracción no paralelizable (1−P) incluye:

  • Lectura de sensores (E/S serial: I²C, SPI, UART)
  • Preprocesamiento de señal (filtrado, FFT)
  • Comunicación con actuadores (PWM, GPIO)

E_total = ∫₀ᵀ P(t) dt = ∑ᵢ Pᵢ · tᵢ Ec. 2.2 — Energía total como suma de modos de operación

Capítulo 03 · Parte I

Fusión de Sensores

Ningún sensor es perfecto. La fusión sensorial combina múltiples fuentes imperfectas para producir una estimación más precisa que la de cualquier sensor individual. Este capítulo presenta los fundamentos matemáticos del Filtro de Kalman y las arquitecturas modernas de fusión neuronal.

3.1 Los Tres Niveles de Fusión

  1. Bajo nivel (datos): combinar lecturas crudas. Ej: múltiples termopares → campo térmico.
  2. Medio nivel (features): combinar características. Ej: GPS + odometría visual → posición.
  3. Alto nivel (decisión): combinar clasificaciones. Ej: voto ponderado entre CNN y detector lidar.

3.2 El Filtro de Kalman: Fundamentos Matemáticos

El Filtro de Kalman es el algoritmo óptimo (en sentido de mínimo error cuadrático medio) para estimar el estado de un sistema lineal con ruido gaussiano.

3.2.1 Modelo del sistema

xₖ = F xₖ₋₁ + B uₖ + wₖ, wₖ ~ 𝒩(0, Q) Ec. 3.1 — Ecuación de estado
zₖ = H xₖ + vₖ, vₖ ~ 𝒩(0, R) Ec. 3.2 — Ecuación de medida

3.2.2 Algoritmo en dos fases

Fase de Predicción (Predict):

x̂ₖ|ₖ₋₁ = F x̂ₖ₋₁|ₖ₋₁ + B uₖ Ec. 3.3 — Predicción del estado
Pₖ|ₖ₋₁ = F Pₖ₋₁|ₖ₋₁ Fᵀ + Q Ec. 3.4 — Predicción de la covarianza

Fase de Actualización (Update):

ỹₖ = zₖ − H x̂ₖ|ₖ₋₁ Ec. 3.5 — Innovación (residuo)
Sₖ = H Pₖ|ₖ₋₁ Hᵀ + R Ec. 3.6 — Covarianza de la innovación
Kₖ = Pₖ|ₖ₋₁ Hᵀ Sₖ⁻¹ Ec. 3.7 — Ganancia de Kalman
x̂ₖ|ₖ = x̂ₖ|ₖ₋₁ + Kₖ ỹₖ Ec. 3.8 — Actualización del estado
Pₖ|ₖ = (I − Kₖ H) Pₖ|ₖ₋₁ Ec. 3.9 — Actualización de la covarianza 
C++17// Filtro de Kalman para fusión IMU (acelerómetro + giroscopio)
#include <array>
#include <cmath>

class KalmanPitch {
    double Q_angle  = 0.001;
    double Q_bias   = 0.003;
    double R_measure = 0.03;

    double angle = 0, bias = 0;
    double P[2][2] = {{0,0},{0,0}};

public:
    double update(double newAngle, double newRate, double dt) {
        double rate = newRate - bias;
        angle += dt * rate;

        P[0][0] += dt * (dt*P[1][1] - P[0][1] - P[1][0] + Q_angle);
        P[0][1] -= dt * P[1][1];
        P[1][0] -= dt * P[1][1];
        P[1][1] += Q_bias * dt;

        double S = P[0][0] + R_measure;
        double K0 = P[0][0] / S;
        double K1 = P[1][0] / S;

        double y = newAngle - angle;
        angle += K0 * y;
        bias  += K1 * y;

        double P00_temp = P[0][0];
        double P01_temp = P[0][1];

        P[0][0] -= K0 * P00_temp;
        P[0][1] -= K0 * P01_temp;
        P[1][0] -= K1 * P00_temp;
        P[1][1] -= K1 * P01_temp;

        return angle;
    }

    double getAngle() const { return angle; }
    double getBias()  const { return bias; }
};

3.3 Fusión Moderna con Atención Multimodal

Los enfoques contemporáneos usan arquitecturas Transformer que aprenden a ponderar cada sensor dinámicamente:

Attention(Q, K, V) = softmax(QKᵀ / √dₖ) · V Ec. 3.10 — Mecanismo de atención (Vaswani et al., 2017)

Donde Q, K, V son proyecciones lineales de las features de cada sensor. La red aprende que, de noche, debe confiar más en el lidar; de día con sol, en la cámara.

💡 Concepto Clave

Atención sensorial adaptativa: similar a cómo el cerebro humano pondera la visión sobre el oído en un entorno iluminado, pero el oído sobre la visión en la oscuridad, una red neuronal multimodal aprende a asignar pesos dinámicos a cada modalidad según su confiabilidad instantánea.

Descubrimiento Acelerado de Materiales

La IA ya no solo analiza materiales: los diseña. AlphaFold revolucionó la biología estructural; GNoME de Google DeepMind ha predicho 2,2 millones de nuevos cristales estables. Este capítulo presenta los fundamentos de la ciencia de materiales y cómo el Machine Learning acelera su descubrimiento.

4.1 Fundamentos de Ciencia de Materiales

Siguiendo a James Newell, las propiedades de un material están determinadas por su estructura, que a su vez depende de su procesamiento:

Propiedades = f(Estructura) = f(g(Procesamiento)) Ec. 4.1 — Tetraedro de la ciencia de materiales

4.1.1 Estructura cristalina

Los metales se organizan en redes cristalinas. Las más comunes son:

EstructuraSímboloÁtomos/celdaFactor empaquetamientoEjemplos
Cúbica centrada en carasFCC40.74Al, Cu, Au, Ni
Cúbica centrada en cuerpoBCC20.68Fe-α, W, Cr
Hexagonal compactaHCP60.74Ti, Mg, Zn
Cúbica simpleSC10.52Po

4.1.2 Defectos cristalinos

Los defectos controlan las propiedades mecánicas. Según Newell:

  • Defectos puntuales: vacantes, intersticiales, sustitucionales
  • Defectos de línea: dislocaciones (borde, tornillo, mixtas)
  • Defectos de superficie: fronteras de grano, superficies libres
  • Defectos de volumen: poros, inclusiones, grietas

4.2 Propiedades Mecánicas Fundamentales

4.2.1 Ley de Hooke generalizada

σᵢⱼ = Cᵢⱼₖₗ · εₖₗ Ec. 4.2 — Ley de Hooke tensorial, C es el tensor de rigidez de 4º orden

Para materiales isótropos, se reduce a dos constantes independientes (módulo de Young E y coeficiente de Poisson ν):

E = σ/ε, ν = −ε_trans / ε_axial Ec. 4.3 — Módulo de Young y coeficiente de Poisson

4.2.2 Criterio de Von Mises (energía de distorsión)

σ_v = √( ½[(σ₁−σ₂)² + (σ₂−σ₃)² + (σ₃−σ₁)²] ) Ec. 4.4 — Esfuerzo equivalente de Von Mises

El material cede cuando σ_v ≥ σ_y (límite elástico).

5.2 Nanotubos de Carbono como Sensores

Los CNT tienen una propiedad extraordinaria: su resistencia eléctrica cambia al deformarse. El factor de galga (gauge factor) es:

GF = (ΔR / R₀) / ε Ec. 5.2 — Factor de galga de un sensor de CNT

Mientras que las galgas metálicas tienen GF ≈ 2, los CNT pueden alcanzar GF > 1000, siendo 500 veces más sensibles.

5.3 Fibras Ópticas FBG (Fiber Bragg Grating)

Una FBG es una variación periódica del índice de refracción en el núcleo de una fibra óptica. La longitud de onda de Bragg es:

λ_B = 2 · n_eff · Λ Ec. 5.3 — Condición de Bragg, Λ es el período de la rejilla

Cuando la fibra se deforma o cambia su temperatura, λ_B se desplaza:

Δλ_B / λ_B = (1 − p_e) · ε + (α + ξ) · ΔT Ec. 5.4 — Sensibilidad a deformación y temperatura

Donde p_e ≈ 0.22 es el coeficiente fotoelástico efectivo, α el coeficiente de expansión térmica y ξ el coeficiente termo-óptico.

5.4 El Ala de Avión que se Siente a Sí Misma

Los composites de fibra de carbono modernos incorporan redes de CNT entre capas. Cuando el ala se flexiona en vuelo, la red cambia su resistencia. Una IA interpreta esos cambios para estimar:

  • La carga estructural en tiempo real
  • El inicio de micro-fisuras (mantenimiento predictivo)
  • La deformación aerodinámica exacta

Diseño Generativo y Geometría Cognitiva

El ingeniero ya no dibuja líneas: escribe restricciones. La IA dibuja la geometría óptima. Este capítulo une el Dibujo Industrial clásico (UNE-EN ISO) con los algoritmos modernos de optimización topológica.

7.1 Del Dibujo Industrial al Diseño Generativo

El dibujo industrial clásico se basa en normas UNE-EN ISO:

  • UNE-EN ISO 5456-2: Proyecciones normalizadas (primer y tercer diedro)
  • UNE-EN ISO 129-1: Principios generales de acotación
  • UNE-EN ISO 1101: Tolerancias geométricas
  • UNE-EN ISO 5455: Escalas
  • UNE-EN ISO 128-20: Tipos de líneas y grosores

Estas normas definen el lenguaje de la ingeniería. Pero el contenido geométrico ahora lo genera la IA.

7.2 Optimización Topológica: Método SIMP

El método SIMP (Solid Isotropic Material with Penalization) es el algoritmo más usado en diseño generativo. Divide el espacio de diseño en una malla de elementos finitos y asigna a cada elemento una "densidad relativa" ρₑ ∈ [0, 1]:

K(ρ) u = f Ec. 7.1 — Ecuación de equilibrio con rigidez dependiente de la densidad
E(ρₑ) = ρₑᵖ · E₀ Ec. 7.2 — Interpolación SIMP, p ≈ 3 es el penalizador

El problema de optimización es:

min_ρ C(ρ) = fᵀ u s.a. ∑ₑ ρₑ · vₑ ≤ V*, 0 < ρ_min ≤ ρₑ ≤ 1 Ec. 7.3 — Minimizar cumplimiento con restricción de volumen

7.3 Tolerancias ISO Aplicadas al Diseño Generativo

El Dibujo Industrial clásico (UNE 1120-96, ISO 406-1987) define tolerancias mediante el sistema ISO de ajustes:

Ajuste = Posición (letra) + Calidad (número) Ec. 7.4 — Sistema ISO de tolerancias, ej: H7/g6

En el diseño generativo, las tolerancias se reinterpretan:

  • Tolerancias deterministas: ±0.1 mm (impresión FDM estándar)
  • Tolerancias probabilistas: distribución normal 𝒩(μ, σ²)
  • Tolerancias generativas: la IA garantiza estadísticamente el cumplimiento
🔮 Insight

Las formas generadas por IA parecen huesos, raíces o ramas. No es casualidad: la evolución biológica también optimizó topologías bajo restricciones físicas durante millones de años. La optimización topológica es "evolución comprimida en horas".

10.1 La Trampa de Python en Sistemas Críticos

Python tiene problemas fundamentales en sistemas embebidos y de tiempo real:

  • GC no determinista: el recolector de basura puede pausar la ejecución en el peor momento (stop-the-world).
  • Overhead de intérprete: latencia de microsegundos inaceptable en control de motores.
  • Consumo de memoria: un runtime de Python puede consumir 50 MB; un MCU tiene 256 KB.
  • Falta de determinismo temporal: imposible garantizar latencias máximas (WCET).

Como explica Stroustrup, C++ combina abstracción de alto nivel con control de bajo nivel:

Eficiencia de C + Abstracción de alto nivel = C++ Ec. 10.1 — Filosofía de diseño de C++

10.2.1 RAII: Resource Acquisition Is Initialization

El principio RAII garantiza que ningún recurso se fuga, incluso ante excepciones:

C++17// RAII aplicado a un sensor IMU
class IMU_Sensor {
    I2C_HandleTypeDef* hi2c;
    bool initialized = false;

public:
    explicit IMU_Sensor(I2C_HandleTypeDef* h) : hi2c(h) {
        init_sensor(hi2c);
        initialized = true;
    }

    ~IMU_Sensor() {
        if (initialized) power_down(hi2c);
    }

    IMU_Sensor(const IMU_Sensor&) = delete;
    IMU_Sensor& operator=(const IMU_Sensor&) = delete;

    IMU_Sensor(IMU_Sensor&& other) noexcept {
        hi2c = other.hi2c;
        initialized = other.initialized;
        other.initialized = false;
    }
};

10.2.2 Templates y Metaprogramación

Los templates permiten escribir código genérico sin overhead en tiempo de ejecución:

C++17// Red neuronal feed-forward con templates (zero-cost abstraction)
template<typename T, size_t In, size_t H1, size_t H2, size_t Out>
struct MLP {
    std::array<T, In*H1>  w1;
    std::array<T, H1>     b1;
    std::array<T, H1*H2>  w2;
    std::array<T, H2>     b2;
    std::array<T, H2*Out> w3;
    std::array<T, Out>    b3;

    constexpr std::array<T, Out> forward(const std::array<T, In>& x) const {
        auto h1 = relu(matmul<In,H1>(x, w1, b1));
        auto h2 = relu(matmul<H1,H2>(h1, w2, b2));
        return softmax(matmul<H2,Out>(h2, w3, b3));
    }
};

// Uso: MLP<float, 6, 32, 16, 3> gesture_model;

C++17// Nodo ROS 2 para procesamiento de imagen con zero-copy
#include "rclcpp/rclcpp.hpp"
#include "sensor_msgs/msg/image.hpp"

class VisionNode : public rclcpp::Node {
    rclcpp::Subscription<sensor_msgs::msg::Image>::SharedPtr sub_;
    rclcpp::Publisher<sensor_msgs::msg::Image>::SharedPtr pub_;
    torch::jit::script::Module model_;

public:
    VisionNode() : Node("vision_node") {
        sub_ = create_subscription<sensor_msgs::msg::Image>(
            "/camera/image_raw", 10,
            [this](const auto& msg) { image_callback(msg); });
        pub_ = create_publisher<sensor_msgs::msg::Image>(
            "/vision/detections", 10);
        model_ = torch::jit::load("yolo_nano.pt");
    }

private:
    void image_callback(const sensor_msgs::msg::Image& msg) {

                      const_cast<uint8_t*>(msg.data.data()));

        auto detections = run_inference(frame);
        publish_detections(detections);
    }
};

12.1 El Arte de lo Imposible

Un modelo de ResNet-50 ocupa 100 MB. Un Arduino Nano tiene 2 KB de RAM. ¿Cómo es posible? Mediante:

12.1.1 Cuantización

Reducir los pesos de 32 bits (float32) a 8 bits (int8):

q = round(x/s) + z Ec. 12.1 — Cuantización asimétrica, s es la escala y z el zero-point

Esto reduce el tamaño del modelo en 4× y acelera la inferencia (las operaciones INT8 son más rápidas que FLOAT32 en MCUs sin FPU).

12.1.2 Pruning

Eliminar conexiones con pesos cercanos a cero:

wᵢⱼ = { wᵢⱼ si |wᵢⱼ| > τ, 0 en otro caso } Ec. 12.2 — Pruning por magnitud con umbral τ

12.1.3 Knowledge Distillation

Un modelo pequeño (estudiante) aprende de uno grande (profesor):

ℒ = α · CE(y, ŷₛ) + (1−α) · T² · KL(pₜ || pₛ) Ec. 12.3 — Loss de distillation (Hinton et al., 2015)

constexpr int kTensorArenaSize = 60 * 1024;
alignas(16) uint8_t tensor_arena[kTensorArenaSize];

tflite::ErrorReporter* error_reporter = nullptr;
const tflite::Model* model = nullptr;
tflite::MicroInterpreter* interpreter = nullptr;

void setup() {
    static tflite::MicroErrorReporter micro_error_reporter;
    error_reporter = &micro_error_reporter;

    model = tflite::GetModel(gesture_model_tflite);
    static tflite::AllOpsResolver resolver;
    static tflite::MicroInterpreter static_interpreter(
        model, resolver, tensor_arena, kTensorArenaSize, error_reporter);
    interpreter = &static_interpreter;

    interpreter->AllocateTensors();
}

void loop() {
    float input_data[128 * 3];
    capture_imu_window(input_data);

    TfLiteTensor* input = interpreter->input(0);
    memcpy(input->data.int8, input_data, sizeof(input_data));

    interpreter->Invoke();

    TfLiteTensor* output = interpreter->output(0);
    int8_t wing_score = output->data.int8[0];
    int8_t ring_score = output->data.int8[1];
    int8_t slope_score = output->data.int8[2];

    if (wing_score > 100) actuate_wing();
    else if (ring_score > 100) actuate_ring();
    else if (slope_score > 100) actuate_slope();
}

13.4 Código de Control en C++ (Arduino)

C++17// Control PID + fusión sensorial + navegación
class RobotController {
    PID pid_left_{2.0, 5.0, 0.1};
    PID pid_right_{2.0, 5.0, 0.1};
    KalmanPitch kalman_pitch_;
    Encoder enc_l_, enc_r_;
    IMU imu_;
    Lidar lidar_;

public:
    void update(float target_v, float target_omega) {
        auto [ax, ay, az, gx, gy, gz] = imu_.read();
        float pitch = kalman_.update(accel_to_pitch(ax, az), gx, 0.01);
        auto distances = lidar_.scan();

        auto [wl, wr] = diff_to_wheel_speeds(target_v, target_omega);

        float ul = pid_left_.compute(wl, enc_l_.velocity());
        float ur = pid_right_.compute(wr, enc_r_.velocity());

        motor_l_.set_pwm(ul);
        motor_r_.set_pwm(ur);

        publish_odometry(enc_l_, enc_r_, pitch);
        publish_lidar_scan(distances);
    }
};
